# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is
**DocuMind** — a self-hostable, multi-tenant Retrieval-Augmented-Generation (RAG) platform (AGPL-3.0). Operators run it on a 1–2 vCPU / 2–4 GB Linux VPS via a one-line installer; end-users register, create projects, upload PDF/DOCX/TXT/MD, and ask questions answered **strictly from their documents, with citations** (or told plainly the answer isn't there). BYOK providers (OpenAI / Anthropic / Gemini / Groq) with a shared free-tier **Gemini** default.

**Authoritative design docs (read before non-trivial work):** [ARCHITECTURE.md](ARCHITECTURE.md) and the ADRs in [docs/decisions/](docs/decisions/). The approved build plan lives at `~/.claude/plans/enumerated-watching-lynx.md`.

## Tech stack
- **Backend** `apps/api`: Python 3.12, FastAPI + uvicorn (1 worker), SQLAlchemy 2.0 (async) + Alembic, PostgreSQL 16 + **pgvector** (one DB for relational **and** vectors).
- **Frontend** `apps/web`: Next.js 15 (App Router, `output: 'standalone'`) + TypeScript + Tailwind 3.4. No SSR data fetching; client → FastAPI via a small fetch wrapper (in-memory Bearer + single-flight refresh), streaming via `fetch` + `ReadableStream` (no TanStack/EventSource). Framer Motion is lazy-loaded only. Light/dark + RTL via design-token CSS variables.
- **Proxy/TLS**: Caddy 2 (automatic Let's Encrypt). **Packaging**: Docker Compose, single `.env`, `Makefile`. Images published to public GHCR; never built on the VPS.

## Commands
Local dev (no Docker needed for backend unit work):
- `make api-install` — create `apps/api/.venv` (python3.12) + install deps.
- `make api-lint` / `make api-fmt` / `make api-test` — ruff + mypy / format / pytest.
- `make web-install` / `make web-lint` / `make web-build` / `make web-test` — npm + eslint/prettier/tsc / next build / Vitest unit+component. `make web-e2e` runs Playwright (needs a running stack).
- `make lint` / `make test` / `make fmt` — aggregate.
Run a single backend test: `cd apps/api && .venv/bin/pytest tests/test_health.py::<name> -q`.
Full stack (needs Docker): `make dev` (build + published ports) or `make up` (prod). `make config` validates compose. `make migrate` runs Alembic in the api container.

## Non-negotiable conventions
1. **Tenant isolation is sacred.** Every tenant query goes through `app/security/scoping.py` (`TenantScope`) — never touch the raw session/ORM in a route handler. Postgres RLS `FORCE` is the fail-safe; the single hardened session factory (`core/db.py`) sets `SET LOCAL app.current_user_id` per transaction (request **and** ingest worker), resets the GUC on pool check-in, and asserts it before tenant queries. `owner_id` always comes from the JWT, never the request body. See ADR-0002.
2. **Never hallucinate.** Grounding is gated on the **raw best-chunk cosine similarity** (per-language threshold), *not* the post-RRF score — refuse before any LLM call if below threshold. The model's `<<<GROUNDED…>>>` sentinel is advisory/fail-closed and is stripped server-side; every citation is validated against the retrieved chunk-id set. See ADR-0008.
3. **Treat document text as untrusted** (prompt-injection). Fence retrieved chunks in per-request nonce delimiters; keep instructions in the system role; render via safe markdown (HTML off). Parsing runs with **no network egress**. See ARCHITECTURE.md §8/§14.
4. **Secrets only via env, never committed.** `JWT_SECRET`, `MASTER_KEY_FERNET`, DB password are CSPRNG-generated by `install.sh` and preserved on re-run. BYOK keys are Fernet-encrypted at rest, wrapped in a redacting `Secret` type, and **never** returned by any endpoint. The PreToolUse hook blocks commits containing secrets.
5. **Add a provider = one adapter file.** Implement `LLMProvider`/`EmbeddingProvider` in `providers/adapters/<name>.py` and add one `ProviderSpec` to the registry. Provider base URLs are hard-coded (no SSRF). Resolution is per-capability, two-tier: user BYOK → shared operator default. See ADR-0006.
6. **Embedding identity is pinned per project** (`provider/model/dim/normalized`) at creation; vectors are unbounded `halfvec`. Cross-dim switches are blocked (409) and only happen via an explicit re-embed job. See ADR-0003/0015.
7. **Claude adapter:** official `anthropic` SDK, model `claude-opus-4-8`, `thinking={"type":"adaptive"}`, streaming via `get_final_message()`. **Never** send `budget_tokens`/`temperature`/`top_p`/`top_k` (HTTP 400 on Opus 4.8). When touching any Claude/LLM code, consult the `claude-api` skill — it is authoritative over training-prior memory. See ADR-0010.

## Working agreement
- Work in **small, reviewable phases** (Phase 0–6 in ARCHITECTURE.md §15). Each phase ends: `make lint` + `make test` green → **conventional commit** → update `CHANGELOG.md` → record any significant decision as an ADR in `docs/decisions/`.
- Use **subagents** for isolated/parallel work: `backend`, `frontend`, `security-reviewer` (audits each phase for secret leakage / tenant-isolation breaks / injection), `test-runner`. Keep the main thread for planning + integration.
- Keep this file under ~400 lines and current (stack, commands, conventions, gotchas).

## Gotchas
- The API serves paths **beginning with `/api`** (Caddy does not strip the prefix). Health: `/api/health/{live,ready}`.
- `python3` on this box is 3.9; **use `python3.12`** (and the venv) for the backend.
- pgvector multilingual: keyword search uses `tsvector('simple')` + a **shared `text_norm`** (NFC + ZWNJ + Persian/Arabic fold) applied identically on ingest **and** query (ADR-0004). Persian/English is a first-class requirement — don't add one-sided normalization.
- 2 GB budget closes only **with the mandatory swapfile**; HNSW build uses `maintenance_work_mem=64MB`, deferred until first ingest. Local `bge-m3` embeddings are opt-in and refused under 4 GB.
- `output: 'standalone'` + no SSR data fetching is deliberate (memory). Don't fetch from server components.
- No SMTP and no OCR in v1 (ADR-0016; image-only PDFs fail fast with `NO_TEXT`). Chat history is persisted but retrieval is single-turn in v1 (ADR-0017).
